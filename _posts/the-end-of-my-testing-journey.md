# 从测试转岗到开发

本人从事了大约4年时间的测试，有幸成为**公司（简称 E 公司）第一个测试工程师，并组建了第一个测试团队，当然这没有什么好炫耀的，主要是让我的读者了解下上下文，毕竟前面写的文章都是跟开发技术相关。

这样的文章，用许倬云先生发明的词就是：心路历程，是的，这篇文章就是分享下我的心路历程。

这篇文章本应该是在 2015 年底就应该写的，那时我刚转岗到另外一个部门做开发岗位，为什么没有在当时写呢？因为那时候我的知乎专栏[一个学渣的程序员生涯](https://zhuanlan.zhihu.com/coding-your-life) 还没开，😄，原因后面再说，先从我进入测试这个领域开始。

## 进入软件测试领域

本人是毕业于一所双非学校（名字就不提了，不给母校丢脸）的软件工程专业，一名名副其实的学渣，大学里面除了专业课学习没干以外其它全干了：泡网吧、玩游戏、踢足球、打篮球，所以成绩完全满足学渣的标准，60分万岁（其实我的GPA是70分，我都没想到），幸好熬到了毕业，这得感谢我的大学老师们，没有他们的仁慈，毕业那将是一件对于我来讲非常困难的事情，怎么个难呢，举个例子：大学期间，我唯一掌握的排序算法就是冒泡排序，那还是为了应付考试，那时候的我根本没有想到我会走上软件开发这条路，因为那时我觉得 c++ 太难了，高数太难了，尤其还要学高数三，最不想去的就是教室了，我看到老师就烦，老师看到我也烦（你要知道一个学渣在学校里面是没有地位的）。

终于熬到了毕业，得找工作，我这样的学渣肯定不会选择做专业方向的工作，但是我又不是什么**代，我当时的目标不是赚钱孝敬爸妈，而是养活自己，生存下去，为了这个目标，我找了一份销售的实习工作，这段经历就隐去，一跟主题无关，二怕读者看完会产生怜悯，毕竟那是一段非常艰辛的经历，后来痛定思痛，觉得个人的能力和性格实在是无法满足一个优秀销售的要求，就想到了做软件测试，那时候觉得自己干软件开发是肯定不行的（现在想想，干软件开发的比我不行的大有人在），那就做软件测试吧。

### Q 公司

经过了一番投简历面试，找到了一家在苏州的台企（简称 Q 公司）的测试工作，公司还算有名，而且岗位还在研发中心（ODM，不是OEM），但是薪水真。。。当然我应该心怀感恩。但是工作可没我想象的那么简单，当时要做的是公司代工的 android 手机蓝牙、Wi-Fi 测试，得看 spec 写测试用例，看过蓝牙、Wi-Fi 协议的人就应该知道，那是多么厚，而且是英文，且到处都是专业术语，我不得不买了几本类似的中文书看，还好我遇到了我迄今为止最好的 mentor，那个时候因为我的 mentor 太好，以至于我都直接称呼她师傅，但是最后面我还是令她失望了，由记得她耐心的跟我讲解：怎么写测试用例、怎么写测试计划、怎么写测试报告、项目的研发流程等等，大学老师都没有像她一样耐心的教导我，我也用我的努力工作去回馈她，从那时候起我就养成了很早就到公司的习惯（当然不能说最早，会被打脸，但是占比较高），很晚从办公室回宿舍，因为我知道我什么都不会，我只有自己的努力和勤奋，陆陆续续的我接到了一些具体的工作，第一次出差，第一次坐飞机，去外地做 field test，坐公交从起始站坐到终点站，然后循环坐，换线路，拿着公司的样机，记录各个运行商的测试数据，遇到问题 dump 出异常的日志，回去上传报 bug。虽然在这家公司呆的时间不久，但是回想起来，收获很多，积累了测试相关技术基础。

### M 公司

离开 Q 公司后我就决定前往魔都上海找工作，借宿在同学那边，住在一个不到 10 平米的阁楼，是的，没错，就是阁楼，且是那种无法直立的阁楼，但是房租便宜，这对我非常重要，然后就开始了有一段充满挫败感的找工作的经历，最终还是在一家外企（简称 M 公司）找到了一份测试的工作，但是是外包，幸运的是，我所外包的部门是原 Intel 在上海的手机芯片部门，M 公司收购了这个部门，正式员工都是名校毕业的硕士、博士，我当然就没有任何其它问题，对我来讲，能在一家公司得到学习成长是我最在乎的，即便是外包，即便薪水很低，由于做的是手机芯片，当然技术就比较集中，难度就稍微高了点，这是我第一次接触 Linux，准确来讲是 Ubuntu，因为办公电脑的系统就是 Ubuntu，而我们的测试产品是开发板，就是那种集中线路板，上面有公司的芯片产品，当然在这个开发板上面跑的依然是 android，那个时候，学会了写 shell 脚本，学会了怎么通过 uboot 烧录系统（行业话就是烧板子），怎么跑 android 的 QTP 测试，对 android 系统有了更深的了解，当然也遇到了对我影响最深的 leader： L 先生，在我眼里，他是一位谦逊、充满好奇心、好学、智慧的人（毕业于上海交大，当然并不是所有上海交大毕业的都这样），芯片研发，涉及到的技术非常的底层，要做到了解都需要花费很大的功夫，要做到理解就非常难了，尤其是对于非本专业的人，但是他总是充满好奇，不时的跟同事讨论起具体的相关技术，开会开着开着就变成了一场技术讨论，像我这样水平的完全插不上话，听的云里雾里，他的能力当然不仅这些，最重要的是他的 leadership，他不是那种只会发号施令的那种，有时他是一个专业人员，有时他又变成你的 mentor 指导你，也会间隔的找你沟通了解你的想法，我十分的佩服，多年后走向管理岗的我由衷的佩服，到现在为止依然如此，还记得他的口头禅：要有这样的 sense，在我眼里，一个 leader 就应该像他一样：专业、富有同理心、热于了解和帮助下属，有一次他找我 one one 的时候，我提出了我想做自动化测试这块，希望他能给我这样的机会，他当时了解了我的想法之后就允诺了如果有机会会找到我，我当时非常感激，到现在依然如此，幸运的是，过了一段时间，从事自动化测试的一位同事要离职，机会就来了，我就被安排进了那个做自动化测试的组里面，还给我指定了 mentor，那个时候我们为了做自动化，搭建了一个 lab，里面都是主机，接着很多块板子，我负责维护一个自动化的核心组件，一个基于 ant 的 Java 项目，是的，你没听错，Java 项目，那个时候的我没有半点 Java 相关的技术积累，都是现学的，还好有个 mentor（一个浙大硕士）帮助我，我也非常努力，但是由于在技术积累方面实在少，短时间内没有什么成就，唯一的成就就是实现了全自动化执行，从 build android 系统版本到自动烧录版本到开发板，再到执行测试用例生成测试报告发送给相关人员，整个流程实现了不需要人工介入，但是流程不是很稳定，因为系统和板子都不一定很稳定，后来由于部门调整以及我的 mentor 离职，后面不久我也离开了 M 公司。

### C 公司

离开 M 公司之后，我有一段时间处于无业游民状态，没错，我是从 M 公司裸辞的，期间也投递了很多公司，面了一些公司（展讯、Intel、三星等），最终在14年年初的时候拿到了 C 公司的 offer，C 公司是国内在线旅游领域市场占有率第一的公司，拿到 offer 后，我很兴奋，我终于进入一家国内一线互联网公司了，

但真实的原因是我并没有想转开发（单指传统的开发，不包括测试开发），缘起我的转岗，而我转岗是想去开发一个测试框架，一个集成的测试平台，用于提升测试同学的测试效率，不幸的是，后面我的 leader 跟我讲这个坑已经被占，且不允许在他的组里面开这样的项目，而那个时候我都已经招聘好了我的接替我的人，
